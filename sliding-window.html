The Two Arrays: Adaptive Threshold

How Dead Signal learns your tapping speed using two sliding windows

The Problem

In morse code, a dit (short) is 1 unit and a dah (long) is 3 units. But "1 unit" means different things to different people — 80ms for a fast tapper, 300ms for a beginner. We need to figure out where your boundary is, and keep updating it as your speed changes.

The Solution: Two Sliding Windows

We maintain two separate arrays, each holding the last 20 durations:

ditDurations[] — last 20 short presses (ms)
avg: --
dahDurations[] — last 20 long presses (ms)
avg: --

The Threshold Calculation

The boundary between dit and dah:
threshold = (avg_dit + avg_dah) / 2

Why a Sliding Window?

Each array holds a maximum of 20 entries. When a 21st entry arrives:

[80, 90, 85, 100, 75, 95, 88, 82, 92, 78,
 86, 91, 84, 79, 93, 87, 81, 96, 83, 89] ← 77 (new)

The oldest entry gets evicted (array.shift()), and the new one is appended (array.push()). This means:

• The average always reflects recent behavior, not ancient history
• If you speed up, the threshold drops within a few taps
• If you slow down, the threshold rises just as quickly
• 20 entries is the sweet spot — enough to smooth out noise, small enough to adapt fast

The Three Phases

Cold Start — fewer than 2 dits or 0 dahs

Uses hardcoded threshold of 200ms. Classification is a guess. Every first-time user starts here.

Calibrating — 2+ dits and 1+ dah collected

Threshold starts adapting. Still jumpy with few samples. The midpoint formula kicks in: (avg_dit + avg_dah) / 2.

Calibrated — 5+ samples in each array

Threshold is stable and accurate. Smoothly tracks speed changes. WPM calculation becomes reliable.

2. Adaptive Threshold

The hardest problem: everyone taps at different speeds. A "long" press for a fast tapper might be shorter than a "short" press for a slow one. The system adapts by tracking your actual timing.

Algorithm — Rolling Average Midpoint

Two sliding windows (max 20 samples each) track recent dit and dah durations. The threshold is the midpoint between their averages:

threshold = (avg_dit + avg_dah) / 2

This means the boundary automatically shifts as you speed up or slow down. Initial threshold is 200ms until enough samples are collected.

3. Gap Detection (Timeouts)

Morse code uses silence to separate letters and words. But how long is "silence"? It depends on the sender's speed.

Algorithm — Proportional Timeouts

Standard morse timing is proportional to the dit length:

letter gap = 3 × avg_dit
word gap = 7 × avg_dit

After each release, two timers start. If you tap again, both cancel. If you don't:

• After 3× dit → commit the current buffer as a letter
• After 7× dit → insert a space (word boundary)

4. Morse Lookup (Hash Map)

Once a letter is committed, the dot-dash string is looked up in a hash map. O(1) — instant.

Data Structure — Object/Hash Map

{ ".-": "A", "-...": "B", "-.-.": "C", ... }

54 entries covering A-Z, 0-9, and punctuation. If the pattern doesn't match anything, it shows ? — which is useful feedback for learning.
